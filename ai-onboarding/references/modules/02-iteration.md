# Module 02: Iteration

Teaches that AI's first output is a starting point, not a finished product. Iteration IS the skill.

## When This Fires

- User accepts a bad or mediocre output without pushing back
- User asks "why isn't this better?" after one attempt
- User treats the first draft as final and moves on disappointed
- User says "AI isn't that good" or "it never gets it right"

## The Problem This Solves

Most people who "tried AI and it didn't work" tried it exactly once. They got a mediocre first output, concluded it was useless, and left. They don't realize that the first output is always rough — the value comes from steering it.

## The Approach

### 1. Show Don't Tell

Take their current task (or a recent one from their workflow). Generate a first output. Then — in front of them — push back on it:

- "That's too generic. Make it specific to [their context]."
- "The tone is wrong. Make it [warmer / sharper / more casual]."
- "You missed [specific detail]. Include it and restructure."
- "Do this 3 different ways. I'll pick the best parts of each."

Show the before and after. The improvement is the lesson.

### 2. The Three Iteration Moves

Teach these as muscle memory:

1. **Reject and redirect.** "No — that's not what I meant. Here's what I actually want: [clearer instruction]."
2. **Selective refinement.** "The structure is good but the tone is wrong. Keep the structure, rewrite with [specific tone]."
3. **Competitive generation.** "Give me 3 versions: one formal, one casual, one bold. I'll tell you which works."

### 3. Let Them Practice

After demonstrating, hand the wheel back. Let them iterate on their own task. Coach from the side:

- "What's wrong with that output?"
- "What would make it better?"
- "Try telling it exactly that."

## Completion Criteria

- [x] User has independently iterated on an AI output at least once (not just watched)
- [x] User can articulate what was wrong with a first draft and how they'd fix it
- [x] User no longer accepts the first output without evaluation

## Common Traps

- **Over-explaining the theory.** Don't say "AI is probabilistic so first outputs are..." Just show them. The theory doesn't help.
- **Doing the iteration for them.** The point is they learn to push back. If you refine it, they just watched a magic trick.
- **Perfectionism.** The goal isn't "always iterate 5 times." It's "know when to iterate and how." Some tasks are one-shot fine.
